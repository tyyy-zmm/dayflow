# Dayflow 功能记录

这个文件用于记录 Dayflow 已经开发的功能。后续每次新增、调整或删除功能，都应该同步更新这里，方便回顾产品演进。

## 基础架构

- 前后端结构：Node.js 后端 + 原生前端。
- 后端入口：`server.js`。
- 前端入口：`public/index.html`。
- 前端样式：`public/styles.css`。
- 前端主逻辑：`public/app.js`。
- API 封装：`public/api.js`。
- 日期工具：`public/date.js`。
- 本地任务数据：`data/tasks.json`。
- 本地设置数据：`data/settings.json`。

## 任务管理

- 新增任务。
- 编辑任务。
- 删除任务。
- 完成 / 取消完成任务。
- 支持快速捕获：只输入标题即可先放进收件箱，之后再安排日期和时间。
- 任务字段：
  - 标题
  - 备注
  - 日期
  - 开始时间
  - 结束时间
  - 优先级
  - 标签
  - 重复规则

## 周计划

- 以完整 7 天为一个周计划。
- 周六、周日和工作日同等展示。
- 支持上一周、本周、下一周切换。
- 支持通过日期选择器快速跳转到任意日期所在周。
- 支持整周折叠。
- 支持每天单独折叠。
- 展开状态会保存在本地。
- 顶部显示本周任务数、完成数和完成度。
- 每天展开后，带开始和结束时间的任务会进入“时间安排”时间轴。
- 只有部分时间信息或没有时间信息的任务，会保留在当天普通清单中。

## 周目标

- 每周可以填写一个“这周最重要的是”目标。
- 周目标保存到 `data/settings.json`。

## 收件箱与已完成

- 未安排日期的任务进入收件箱。
- 已完成任务可以通过独立入口查看。
- 当前实现为侧边抽屉。
- 收件箱任务支持快速安排到今天、明天或当前选中日期。

## 筛选与搜索

- 支持任务关键词搜索。
- 支持状态筛选：
  - 全部状态
  - 未完成
  - 已完成
  - 高优先级
- 支持标签筛选。

## 任务移动

- 支持把任务快速移动到下一天。
- 支持拖拽任务到本周任意一天。

## 重复任务

- 新增任务时可选择：
  - 不重复
  - 每天，连续 7 天
  - 每周，连续 4 周

## 数据导入导出

- 支持导出 JSON。
- 支持从 JSON 导入。
- 导出内容包含任务和设置。

## 显示密度

- 支持显示密度切换：
  - 舒适
  - 紧凑
  - 极简

## 视觉与动效

- 使用更清爽的自然色主题。
- 左侧栏、周计划、周目标、任务优先级和抽屉有独立色彩层级。
- 高优先级和中优先级任务有左侧色条。
- 页面进入、每日展开、抽屉打开、任务 hover 有轻量动效。
- 支持系统减少动效偏好。

## 后端 API

- `GET /api/health`：健康检查。
- `GET /api/tasks`：读取任务。
- `POST /api/tasks`：新增任务。
- `PUT /api/tasks/:id`：更新任务。
- `DELETE /api/tasks/:id`：删除任务。
- `GET /api/settings`：读取设置。
- `PUT /api/settings`：更新设置。
- `GET /api/export`：导出任务和设置。
- `POST /api/import`：导入任务和设置。

## 待优化

- 已完成任务入口可以考虑改成独立视图或轻量弹层。
- 拖拽移动需要补充更明显的视觉反馈。
- 重复任务目前是批量创建，后续可以升级为真正的重复规则。
- 时间轴目前是按任务时间顺序分组展示，后续可以升级为更接近日历的纵向比例布局。
